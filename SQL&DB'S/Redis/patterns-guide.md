Redis is an in-memory data structure store used as a database, cache, message broker, and streaming engine.

## Why Redis?
- **Speed**: in-memory storage, sub-millisecond latency
- **Data structures**: strings, hashes, lists, sets, sorted sets, streams
- **Persistence**: optional disk snapshots and append-only file (AOF)
- **Pub/Sub**: lightweight message broadcasting
- **Atomic operations**: increment, decrement, push, pop

## Common patterns
- **Caching**: reduce database load
- **Session storage**: fast user session data
- **Leaderboards**: sorted sets for rankings
- **Rate limiting**: track request counts
- **Pub/Sub**: real-time notifications
- **Queues**: background job processing
