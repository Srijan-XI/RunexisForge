# Remix Usage Guide

## Create a Project
```bash
npx create-remix@latest
# choose TypeScript and a deploy target (Node, Vercel, Cloudflare, etc.)
cd <app>
npm install
```

## Run
```bash
npm run dev
```

## Routing
- File-based under `app/routes/`
- Nested routes: `app/routes/dashboard.tsx` and `app/routes/dashboard.settings.tsx`
- Dynamic params: `app/routes/blog.$slug.tsx`

## Loaders and Actions
```tsx
export async function loader() {
  return json({ message: 'Hello' });
}

export async function action({ request }) {
  const data = await request.formData();
  return redirect('/thanks');
}
```
Use `useLoaderData` and `useActionData` in components.

## Styling
- CSS files imported per route: `import styles from './route.css';`
- Tailwind: `npx tailwindcss init -p` and add to `app/root.tsx`

## Deployment
- Node adapter: `npm run build && npm run start`
- Vercel/Cloudflare use appropriate adapters generated by create-remix

## Testing
- Unit: Vitest/Jest
- E2E: Playwright or Cypress
